# Taste

Use this skill when a YouTrader task changes any visible UI, export, PDF, share card, paywall, empty state, chart, card, or screen copy.

## Taste Rules

- Start from `docs/MY_UI.md`, not generic app design.
- Keep YouTrader premium, dark, institutional, trader-first, and calm.
- Green is for positive/profit/safe buffer.
- Red is for loss/risk/danger.
- Purple is the premium secondary accent.
- Remove clutter before adding decoration.
- Make the trader's next action obvious within 3 seconds.
- Preserve business logic; taste is not an excuse to rewrite flows.

## Workflow

1. Identify the exact screen or artifact being changed.
2. State the user-facing job of that surface in one sentence.
3. Check whether the current change improves hierarchy, readability, spacing, and trust.
4. Fix only obvious issues inside the requested scope.
5. Run `design-review` before completion for UI tasks.

## Avoid

- Cheap crypto/casino visuals.
- Neon overload.
- Duplicate cards or contradictory metrics.
- Tiny unreadable labels.
- Paywalls before value.
- Marketing copy where a trader needs operational clarity.
